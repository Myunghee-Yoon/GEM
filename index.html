프롬프트 엔지니어링의 핵심, PARTS

P

Persona (역할)

AI에게 특정 전문가의 역할을 부여합니다. AI가 어떤 관점에서 생각하고 답변해야 할지 방향을 설정합니다.

예: "당신은 세계 최고의 마케팅 전략가입니다."

A

Action (행동)

AI가 수행해야 할 구체적인 행동을 정의합니다. '작성', '분석', '요약', '비교' 등 명확한 동사를 사용합니다.

예: "아래 주제에 대한 블로그 포스트 초안을 작성해주세요."

R

Request (요청)

수행할 작업의 핵심 내용입니다. 무엇에 대해 행동을 취해야 하는지 구체적으로 명시합니다.

예: "2025년 기술 트렌드"

T

Target (대상)

결과물을 소비할 대상을 지정합니다. 대상에 따라 AI는 어조, 어휘, 설명의 깊이를 조절합니다.

예: "IT 분야에 익숙하지 않은 일반 대중"

S

Style (스타일)

결과물의 전체적인 톤앤매너를 설정합니다. '전문적', '친근한', '유머러스한' 등 원하는 분위기를 지정합니다.

예: "쉽고 재미있게 읽히는 친근한 스타일로 작성해줘."

요청사항 작성하기

'요청사항' 필드에 GEM이 수행할 역할을 상세히 적어줍니다. 아래 예시를 복사하여 사용하거나 자신만의 스타일로 수정해보세요.

당신은 'PARTS 프레임워크'를 기반으로 사용자의 아이디어를 완벽한 프롬프트로 재구성하는 '프롬프트 설계 전문가'입니다. 사용자가 아이디어나 키워드를 제시하면, 아래의 단계를 따라 구조화된 프롬프트를 생성해야 합니다. 1. 사용자의 요청을 분석하여 PARTS(Persona, Action, Request, Target, Style)의 각 요소를 파악합니다. 2. 만약 특정 요소가 부족하다면, 사용자에게 친절하게 질문하여 정보를 보충합니다. 3. 모든 요소가 갖춰지면, 이를 조합하여 명확하고 실행 가능한 최종 프롬프트를 생성합니다. 4. 생성된 프롬프트와 함께, 각 PARTS 요소가 어떻게 적용되었는지 간략하게 설명해줍니다.

Gemini로 요청사항 다듬기

작성한 요청사항을 Gemini를 활용해 더욱 정교하게 다듬을 수 있습니다.
요청사항 입력창 우측의 ✨ 아이콘을 클릭하여 Gemini가 제안하는 개선안을 확인하고 적용해보세요. 더 명확하고 효과적인 지시문으로 개선될 것입니다.
